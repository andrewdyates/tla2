hindex_props.smt2 (P1):
  For sorted array a0..a4, compute's result h satisfies:
  count_ge(a, h) >= h AND count_ge(a, h+1) < h+1.
  Negation is UNSAT => property holds universally.

hindex_unique.smt2 (P2):
  No two distinct values h1 < h2 both satisfy the h-index definition.
  Proof via monotonicity: count_ge(a, h2) <= count_ge(a, h1+1) < h1+1 <= h2,
  giving h2 < h2, contradiction.

hindex_update_bound.smt2 (P3):
  Incrementing one element by 1 cannot increase h-index by 2 or more.
  Uses count_ge monotonicity and the +1 counting bound.

compute_equiv.smt2 (P4):
  For sorted arrays, compute (linear scan) and compute_opt (binary search)
  return the same value. Both find the length of the prefix where a[j] > j.
